## Краткая хронология прошлого чата

1. Обсудили проект "умной игрушки для кота" на ESP32 с компонентами:
   - ESP32-WROOM-32
   - SG90 (360 + 180)
   - VL53L0X
   - SSD1306 (SPI)
   - лазер через транзистор
   - динамик

2. Согласовали пины с учетом OLED по SPI:
   - OLED_MOSI=23, OLED_CLK=18, OLED_DC=2, OLED_CS=5, OLED_RESET=4
   - SERVO_PAN=19, SERVO_TILT=27, LASER=26, SPEAKER=25
   - I2C: SDA=21, SCL=22

3. Был подготовлен скетч с комментариями на русском:
   - FreeRTOS-задачи: sensor, motion, sound, ui
   - режимы: SEARCH, CALM PLAY, ACTIVE PLAY, POWER SAVE

4. Были созданы версии файлов для безопасной доработки:
   - `v1/smart_cat_toy_v1.ino` как зафиксированная версия
   - `v2/smart_cat_toy_v2.ino` как рабочая версия (позже перенесена в sketch-папку)

## Важно помнить

- Для SG90 360 нет обратной связи по абсолютному углу.
- Поэтому абсолютная позиция после перезапуска не гарантируется аппаратно.
- Но текущие ограничения и мягкие шаги заметно снижают риск намотки проводов.

## Техническое исправление компиляции (2026-04-24)

- Ошибка: `ledcSetup was not declared in this scope` при сборке `v2`.
- Причина: различие API ESP32 core 2.x и 3.x.
- Исправление в `v2/smart_cat_toy_v2/smart_cat_toy_v2.ino`:
  - добавлены условные блоки `#if ESP_ARDUINO_VERSION_MAJOR >= 3`
  - для core 3.x: `ledcAttach(...)` и `ledcWriteTone(SPEAKER_PIN, ...)`
  - для core 2.x: `ledcSetup(...)` + `ledcAttachPin(...)` и `ledcWriteTone(BUZZ_CH, ...)`

## Исправление намотки SG90-360 (2026-04-24)

- Симптом: накопительный проворот `pan` > 180 градусов и намотка проводов.
- Причина: SG90-360 не сообщает реальный угол, накопление команд приводит к дрейфу.
- Исправление:
  - удален накопительный виртуальный угол `panVirtualDeg`
  - добавлен безопасный режим `panSafeJitter()` (симметричные импульсы вправо/влево)
  - уменьшена амплитуда команд (`PAN_LEFT_US/PAN_RIGHT_US` ближе к 1500)
  - сокращено время импульса `PAN_STEP_MS` для более тихой и мягкой работы

## Отображение H и усиление pan (2026-04-24)

- По запросу добавлен вывод `H=` на OLED в последней строке.
- `H` = оценка относительного угла горизонтали (SG90-360) от позиции на reset (`0`), в диапазоне `-45..+45` (фактически ограничено `±22` по текущему конусу).
- Чтобы движение по горизонтали ощущалось лучше:
  - увеличена амплитуда команд панорамы (`1470/1530`)
  - увеличена длительность шага панорамы (`PAN_STEP_MS = 60`)
  - добавлен расчет и хранение `panRelDeg` в `SharedState`

## Почему H менялся, а pan "не ехал" (2026-04-24)

- Симптом: на OLED `H` бегает, но физически SG90-360 почти не двигается.
- Причина: `H` — оценочный угол в коде, а мотор мог оставаться в мертвой зоне вокруг `1500us`.
- Исправление:
  - введена явная дельта от стопа: `PAN_DELTA_US = 120`
  - команды панорамы теперь формируются как `1500 ± 120` (обычно уже достаточно для старта 360-сервы)
  - `PAN_STEP_MS` уменьшен до короткого импульса (мягко, но заметно)
  - в `ACTIVE PLAY` включен более "круговой" паттерн движения зайчика

## Добавлен режим калибровки через Serial (2026-04-24)

- Реализованы два режима:
  - `RUN` — обычная игровая логика
  - `CALIB` — тихий режим настройки (движение/звук остановлены)
- Параметры вынесены в RuntimeConfig и сохраняются в `Preferences` (namespace `cattoy`).
- Команды Serial Monitor:
  - `help`
  - `show`
  - `mode run`
  - `mode calib`
  - `set pan_delta <60..220>`
  - `set pan_step <5..120>`
  - `set pan_settle <20..200>`
  - `set cone <8..45>`
  - `set tilt_home <45..120>`
  - `pan_speed slow|normal|fast`
  - `save`
